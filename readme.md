# ЖК «Позитив» — сайт

Сайт жилого комплекса «Позитив»: каталог квартир, новости, галерея, этапы строительства, ипотечный калькулятор, админка.

Работает на Laravel 5.2 / PHP 7.4. Разворачивается через Docker Compose — 4 контейнера (PHP-FPM, Nginx, MySQL, Redis).

## Что нужно для запуска

Вместе с репозиторием передаётся **медиа-архив** (`dom-pozitiv-media.tar.gz`) — картинки, шрифты, документы. Они не лежат в git из-за объёма.

Также нужен **дамп базы данных** (`.sql` файл).

## Как развернуть

```bash
# 1. Склонировать репозиторий
git clone <url> dom-pozitiv
cd dom-pozitiv

# 2. Распаковать медиа-архив в корень проекта
tar xzf dom-pozitiv-media.tar.gz

# 3. Положить SQL-дамп для первичного импорта
cp dump.sql docker/mysql/init/

# 4. Создать файл настроек из шаблона
cp .env.example .env

# 5. Сгенерировать ключ приложения
docker compose run --rm --no-deps app php artisan key:generate

# 6. Отредактировать .env — как минимум:
#    - DB_PASSWORD / MYSQL_ROOT_PASSWORD (пароли БД)
#    - MAIL_* (настройки почты для заявок на бронирование)
#    - GOOGLE_MAPS_API_KEY (карта на странице контактов)
#    - APP_URL (адрес сайта)

# 7. Запустить
docker compose up -d
```

Сайт будет доступен на `http://localhost:8080` (порт настраивается через `HTTP_PORT` в `.env`).

## Что где лежит

**В Docker-образе** (собирается из Dockerfile):
- PHP-код, конфиги, шаблоны, зависимости Composer — всё, что нужно для работы приложения.

**На диске хоста** (монтируется в контейнеры):
- `public/` — статика сайта + медиа из архива. Отдаётся Nginx напрямую.
- `storage/` — логи, кэш, сессии. Пишется приложением.
- `bootstrap/cache/` — скомпилированные конфиги Laravel.
- `docker/mysql/init/` — SQL-файлы для первичного импорта в базу.

**Docker-тома** (данные контейнеров):
- `mysql_data` — файлы базы данных MySQL.
- `redis_data` — данные Redis.

## Медиа-архив

Архив содержит каталоги, которые распаковываются в корень проекта:

| Каталог | Содержимое |
|---------|------------|
| `public/images/` | Фотографии: этапы строительства, новости, галерея |
| `public/assets/` | Графика интерфейса, фото квартир |
| `public/fonts/` | Шрифты сайта |
| `public/documents/` | PDF-документы (проектные декларации и пр.) |
| `public/img/` | Дополнительные изображения |
| `public/media/` | Шрифты ипотечного калькулятора |

Для создания архива (на машине, где есть все файлы):

```bash
tar czf dom-pozitiv-media.tar.gz public/images/ public/assets/ public/fonts/ public/documents/ public/img/ public/media/
```

## Импорт базы данных

SQL-файлы из `docker/mysql/init/` загружаются автоматически при **первом** создании тома `mysql_data`.

Если нужно импортировать заново — удалите том и перезапустите:

```bash
docker compose down
docker volume rm dom-pozitiv_mysql_data
docker compose up -d
```

Данные в базе при этом будут потеряны.

## Настройки (.env)

Основные переменные, которые нужно заполнить:

| Переменная | Что это |
|------------|---------|
| `APP_KEY` | Генерируется командой `key:generate` (шаг 5) |
| `APP_URL` | Адрес сайта (например `https://dom-pozitiv.ru`) |
| `DB_PASSWORD` | Пароль пользователя БД (должен совпадать при первом запуске) |
| `MYSQL_ROOT_PASSWORD` | Пароль root MySQL |
| `MAIL_*` | SMTP-сервер для отправки заявок на бронирование |
| `GOOGLE_MAPS_API_KEY` | Ключ Google Maps (страницы контактов, инфраструктуры) |
| `HTTP_PORT` | Порт сайта на хосте (по умолчанию 8080) |
